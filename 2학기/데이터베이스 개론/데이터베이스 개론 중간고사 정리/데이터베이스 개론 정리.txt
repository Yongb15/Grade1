데이터베이스 개론 서술형

1번 데이터베이스의 정의와 특징

정의
1) 데이터베이스는 공유 데이터이다.

2) 데이터베이스는 통합 데이터이다.

3) 데이터베이스는 저장 데이터이다.

4) 데이터베이스는 운영 데이터이다.

특징
1) 데이터베이스는 실시간 접근이 가능하다.

2) 데이터베이스는 계속 변화한다.

3) 데이터베이스는 동시공유가 가능하다.

4) 데이터베이스는 내용으로 참조가 가능하다

2번 데이터베이스 관리 시스템(DBMS)이 무엇인가?

1) 파일 시스템의 데이터 중복과 데이터 종속 문제를 해결하기 위해 제시된 소프트웨어
2) 조직에 필요한 데이터를 데이터베이스에 통합하여 저장하고 이에 대한 관리를 집중적으로 담당

데이터베이스 관리 시스템의 주요기능

1) ★ 정의 기능 
	ㄴ 데이터베이스 구조를 정의하거나, 이미 정의된 구조를 수정
2) 조작 기능
	ㄴ 데이터를 삽입, 수정, 검색하는 연산을 효율적으로 처리
3) 제어 기능
	ㄴ 데이터를 여러 사용자가 공유해도 항상 정확하고 안전하게 유지하는 기능을 제공

3번 3단계 데이터 베이스 구조 설명

1) 데이터베이스를 쉽게 이해하고 이용할 수 있도록 하나의 데이터베이스를 관점에 따라 세 단계로 나눈 것
	ㄴ 외부 단계 : 개별 사용자 관점
	ㄴ 개념 단계 : 조직 전체의 관점
	ㄴ 내부 단계 : 물리적인 저장 장치의 관점
2) 각 단계별로 다른 추상화 제공
	ㄴ 내부에서 외부 단계로 갈수록 추상화 레벨이 높아짐

4번 데이터 독립성이란?

하위 스키마를 변경하더라도 상위 스키마가 영향을 받지 않는 특성이다.

5번 논리적 독립성, 물리적 독립성

논리적 독립성 : 개념 스키마가 변경되어도 외부 스키마가 영향을 받지 않는 것이다. 응용 인터페이스라고도 한다.
물리적 독립성 : 내부 스키마가 변경되어도 개념 스키마가 영향을 받지 않는 것이다. 저장 인터페이스라고도 한다.

6번 데이터 모델의 구성요소

데이터 구조
- 자주 변하지 않는 정적인 특성
- 개념적 데이터 모델에서는 현실 세계를 개념 세계로 추상화했을 때 어떤 요소로 이루어져 있는지를 표현하는 개념적 구조
- 논리적 데이터 모델에서는 데이터를 어떤 모습으로 저장할 것인지 표현하는 논리적 구조

연산
- 데이터 구조에 따라 개념 세계나 컴퓨터 세계에서 실제로 표한한 값들을 처리하는 작업
- 값이 연산에 의해 계속 변경될 수 있어 동적인 특성을 갖음

제약조건
-구조적 측면의 제약 사항과 연산을 적용하는 경우 허용할 수 있는 의미적 측면의 제약사항이 있음

7번 스키마와 인스턴스
스키마는 데이터베이스에 저장되는 데이터 구조와 제약조건을 정의한 것이다.
인스턴스는 스키마에 따라 데이터베이스에 실제로 저장된 값이다.

9번 차수와 카디널리티
차수는 하나의 릴레이션에서 속성 전체 개수이고,
카디널리티는 하나의 릴레이션에서 투플의 전체 개수이다.

10번 릴레이션의 특성 - 투플과 속성
투플의 유일성 : 하나의 릴레이션에는 동일한 투플이 존재할 수 없다.
투플의 무순서 : 하나의 릴레이션에서 투플 사이의 순서는 무의미하다.
속성의 무순서 : 하나의 릴레이션에서 속성 사이의 순서는 무의미하다.
속성의 원자성 : 속성 값으로 원자 값만 사용할 수 있다.

11번 키의 특성 - 유일성과 최소성
유일성은 하나의 릴레이션에서 모든 투플은 서로 다른 키 값을 가져야한다.
최소성은 꼭 필요한 최소한의 속성들로만 키를 구성해야 한다.

13번 개체무결성 제약조건, 참조 무결성 제약조건
개체무결성 제약조건
